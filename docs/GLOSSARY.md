# Glossary

Ralph-specific vocabulary, in alphabetical order. General software terms (hook, circuit breaker, mutex) are not defined here — look them up elsewhere.

## Adaptive timeout

Percentile-based timeout for Claude CLI invocations that learns from recent run times. Prevents a slow network or model switch from tripping a fixed timeout. Implemented in `sdk/ralph_sdk/circuit_breaker.py` (`ConsecutiveTimeoutDetector`) and the bash equivalent.

## AGENT.md

A file inside `.ralph/` that describes how to build, run, and test the project. Ralph maintains it automatically. Distinct from [AGENTS.md](#agentsmd) which is for AI tool guidance.

## AGENTS.md

Top-level file with guidance for AI assistants working in this repo. Generated by `tapps_init` and kept in sync by `tapps_upgrade`. Not hand-edited.

## ALLOWED_TOOLS

Comma-separated whitelist of Claude Code tools the loop will allow. Includes granular bash patterns like `Bash(git -C *)`, `Bash(grep *)`. Defined in `.ralphrc` or passed via `--allowed-tools`. Deny-by-default: anything not listed is blocked.

## atomic_write

A shell helper in `ralph_loop.sh` that writes to a unique temp path, best-effort fsyncs, then `mv -f`s into place — so a SIGTERM between truncate and write can't leave a zero-byte counter. All counter and state-file writes use it (TAP-535).

## Batch

A group of tasks Ralph processes in one Claude invocation. Up to 8 SMALL / 5 MEDIUM / 1 LARGE per batch (sizes increased in v1.8.5 because epic-boundary QA catches regressions). See [complexity classifier](#complexity-classifier).

## Circuit breaker

Three-state failure-handling pattern in `lib/circuit_breaker.sh`:

- **CLOSED** — nominal operation.
- **OPEN** — halted; waiting for cooldown or manual reset.
- **HALF_OPEN** — cooldown elapsed; next iteration probes.

Transitions tracked in `.circuit_breaker_state`, sliding-window events in `.circuit_breaker_events`. Auto-recovery via `CB_AUTO_RESET=true` bypasses cooldown.

## Completion indicators

A rolling count of "looks done" signals the bash loop derives from Claude's text output (NLP heuristics). One half of the [dual-condition exit gate](#exit-gate).

## Continue-As-New

Temporal-inspired pattern: after `RALPH_MAX_SESSION_ITERATIONS` (default 20) or `RALPH_MAX_SESSION_AGE_MINUTES` (default 120), the session resets while carrying forward essential state (current task, progress, recommendation). Fixes the ~35-min agent-success cliff documented in research. CTXMGMT-3.

## Complexity classifier

5-level task classifier (TRIVIAL → SMALL → MEDIUM → LARGE → ARCHITECTURAL) in `lib/complexity.sh` and `sdk/ralph_sdk/complexity.py`. Feeds into `cost.select_model()` for dynamic model routing. Annotation overrides: `[TRIVIAL]`, `[SMALL]`, `[MEDIUM]`, `[LARGE]`, `[ARCH]` in task names.

## Dry-run

`--dry-run` or `DRY_RUN=true`. Simulates one loop iteration without calling the Claude API. Writes `status.json` with `status: "DRY_RUN"` and exits. Useful for validating configuration and tool permissions.

## Dual-condition exit gate

The core safety property: Ralph exits only when **both** `completion_indicators >= 2` **and** Claude's `EXIT_SIGNAL: true` agree. See [ADR-0001](decisions/0001-dual-condition-exit-gate.md).

## Epic-boundary

The point where Ralph completes the last `- [ ]` task under a `##` section in `fix_plan.md`. Mid-epic iterations are "hot"; boundaries are where expensive operations (QA, backups, explorer re-runs) fire. See [ADR-0004](decisions/0004-epic-boundary-qa-deferral.md).

## EXIT_SIGNAL

Boolean field in the RALPH_STATUS block that Claude sets when it believes work is complete. One half of the [dual-condition exit gate](#exit-gate). Must be `true` **and** agree with the loop's completion-indicator count.

## EXIT-CLEAN

The 4th branch in `on-stop.sh`'s circuit-breaker logic that recognizes `EXIT_SIGNAL: true && STATUS: COMPLETE` as a clean exit even when files/tasks are zero. Prevents the CB from tripping on the same signal Claude uses to ask for shutdown.

## Failsafe

Safe fallback behaviors when components are missing or degraded. Codified in [FAILSAFE.md](../FAILSAFE.md).

## Fast-trip

Circuit-breaker detector that opens immediately (bypassing the sliding window) when it sees consecutive 0-tool-use runs under 30s, consecutive `TESTS_STATUS: DEFERRED`, or consecutive timeouts. SDK-SAFETY-1.

## fix_plan.md

A prioritized checklist of tasks inside `.ralph/`. Ralph checks off `[x]` items as it completes them. You + Ralph both edit it. Replaced by Linear queries when `RALPH_TASK_SOURCE=linear`.

## flock

The Linux `flock(2)` advisory lock, used on `.ralph/.ralph.lock` to prevent concurrent Ralph instances from corrupting shared state (LOCK-1).

## Four-layer rate limit detection

Ralph's defense against API rate limits: (1) timeout exit code 124, (2) structural `rate_limit_event` JSON, (3) filtered text fallback on last 30 lines, (4) Extra-Usage quota detection. Layers 3-4 filter echoed project content to avoid false positives.

## gawk

GNU awk. Required (not mawk) because the plan optimizer (PLANOPT-2) uses `match(s, re, arr)` array capture, which mawk doesn't support.

## grep -c pitfall

A shell gotcha: `count=$(grep -c PAT || echo "0")` produces `"0\n0"` on no-match because `grep -c` exits 1 with stdout `"0"`. Historically this corrupted `status.json`. Fix: pipe through `tr -cd '0-9'` and apply `${var:-0}` before arithmetic. Documented in CLAUDE.md.

## Hive

Memory-sharing layer in tapps-brain. Four scopes: `private` (saver-only), `domain` (same profile), `hive` (all agents), `group:<name>` (named group). Disabled by default; enable via `hive_status` tool. See [RALPH-STACK-GUIDE.md](RALPH-STACK-GUIDE.md).

## Killswitch

Emergency stop mechanism: touching `.ralph/.killswitch` halts the loop at the top of the next iteration with full cleanup. Useful for headless/fleet operations where signal delivery is impractical. See [KILLSWITCH.md](../KILLSWITCH.md).

## Linear backend

Task source mode selected via `RALPH_TASK_SOURCE=linear`. Reads the Linear GraphQL API instead of `fix_plan.md`. Fail-loud on API errors (TAP-536): distinguishes "exit non-zero" (unknown) from "exit 0 + value" (real result). See [LINEAR-WORKFLOW.md](LINEAR-WORKFLOW.md).

## Live stream / `--live`

Real-time streaming of Claude Code output via an awk-based JSONL filter. Shows tool calls with parameters, elapsed time, sub-agent events, errors. Text content filtered for readability: metadata suppressed, long lines truncated, UUID patterns stripped.

## MCP

Model Context Protocol — the protocol Claude Code uses to talk to external tool servers. Ralph probes and injects guidance for three: **tapps-mcp** (quality scoring), **tapps-brain** (memory), **docs-mcp** (doc generation). Projects register MCPs in `.mcp.json`.

## Plan optimizer

The module that reorders unchecked tasks in `fix_plan.md` by dependency before each session (`RALPH_NO_OPTIMIZE=false`). Three-layer dependency detection: (1) AST-based import graph, (2) explicit metadata, (3) phase convention. Toposort via `tsort`, secondary sort by module / phase / size. `lib/plan_optimizer.sh` and `sdk/ralph_sdk/plan_optimizer.py`.

## PROMPT.md

File inside `.ralph/` that holds development instructions Claude reads at the start of every loop. Customized by the project operator — this is where you tell Ralph what you're building.

## Push-mode (Linear)

Fallback when `LINEAR_API_KEY` is unset (OAuth-via-MCP deployments): `linear_get_open_count` / `linear_get_done_count` read from `status.json` written by the stop hook from Claude's RALPH_STATUS block. Entries older than `RALPH_LINEAR_COUNTS_MAX_AGE_SECONDS` (default 900) abstain. TAP-741.

## RALPH_STATUS

Structured status block Claude emits at the end of every response. Parsed by `on-stop.sh` into `status.json`. Fields: `STATUS`, `WORK_TYPE`, `TASKS_COMPLETED`, `FILES_MODIFIED`, `TESTS_STATUS`, `EXIT_SIGNAL`, `RECOMMENDATION`, plus push-mode `LINEAR_OPEN_COUNT` / `LINEAR_DONE_COUNT`.

## ralph-architect

Sub-agent running on Opus for LARGE / ARCHITECTURAL tasks. Mandatory ralph-reviewer pass afterwards. Defined in `.claude/agents/ralph-architect.md`.

## ralph-bg-tester

Background sub-agent that runs tests asynchronously while the main loop continues. Returns results via hook. Avoids blocking throughput on slow test suites.

## ralph-doctor

Diagnostic command that verifies dependencies, hook consistency vs templates, PATH configuration, and reports drift warnings.

## ralph-explorer

Read-only sub-agent running on Haiku for fast codebase search. Skipped for consecutive SMALL tasks in the same module (use Glob/Grep directly instead).

## ralph-reviewer

Read-only sub-agent running on Sonnet for code review. Mandatory before `EXIT_SIGNAL: true` and for LARGE/ARCHITECTURAL tasks.

## ralph-tester

Worktree-isolated sub-agent running on Sonnet for test execution. Skipped mid-epic (set `TESTS_STATUS: DEFERRED`); mandatory at epic boundaries.

## ralph-upgrade / ralph-upgrade-project

`ralph-upgrade` refreshes the global install (`~/.ralph/`, `~/.local/bin/`). `ralph-upgrade-project` propagates template updates to an existing managed project's `.ralph/hooks/` and templates. Run both after a Ralph release.

## Sandbox

`ralph --sandbox` runs the loop inside a Docker container. V2 supports rootless Docker, `--network none`, and gVisor runtime. `lib/sandbox.sh`.

## SDK

Python reimplementation of the Ralph state machine in `sdk/ralph_sdk/`. Full async, Pydantic v2, pluggable state backend. Selectable at runtime via `ralph --sdk`. See [ADR-0005](decisions/0005-bash-sdk-duality.md) and [sdk-guide.md](sdk-guide.md).

## Session ID

A Claude conversation identifier persisted in `.ralph/.claude_session_id` with 24-hour expiration. Passed as `--resume <id>` on each invocation for conversation continuity. Reset on CB open, manual interrupt, or `is_error: true`.

## Stream parser / v2

The `awk`-based filter that converts Claude's JSONL stream into a human-readable live view. "v2" refers to the multi-result-filtering and unescape-RALPH_STATUS rewrite in v1.2.0.

## status.json

The canonical per-iteration state file written by `on-stop.sh`. Everything downstream — the loop, the monitor, SDK wrappers — reads this file instead of the raw stream. Atomic writes via `mv` + `rm -f` fallback for WSL/NTFS.

## TAP-\<number\>

Linear ticket IDs in the `TappsCodingAgents` workspace. Referenced throughout CLAUDE.md and the changelog. Examples: TAP-535 (atomic writes), TAP-536 (Linear fail-loud), TAP-538 (hook resilience), TAP-574/575 (skill baseline), TAP-741 (Linear push-mode).

## tapps-brain

Persistent cross-session memory MCP server. HTTP-based (default `http://127.0.0.1:8080`). Exposes `brain_recall`, `brain_remember`, `brain_learn_success`, `brain_learn_failure`, `brain_forget` tools. Optional Hive layer for multi-agent memory sharing.

## tapps-mcp

Code quality MCP server. Exposes `tapps_quality_gate`, `tapps_score_file`, `tapps_lookup_docs`, `tapps_impact_analysis`, `tapps_checklist`. Project-registered in `.mcp.json`.

## Task source

Selected by `RALPH_TASK_SOURCE`. Two values: `file` (default — reads `fix_plan.md`) and `linear` (reads Linear GraphQL API).

## TESTS_STATUS

RALPH_STATUS field: `PASS` | `FAIL` | `DEFERRED` | `N/A`. `DEFERRED` means tests were intentionally skipped mid-epic; promoted to `PASS`/`FAIL` at epic boundaries when ralph-tester runs.

## WORK_TYPE

RALPH_STATUS field describing the iteration's main activity: `IMPLEMENTATION` | `TESTING` | `REVIEW` | `DEBUG` | `DOCS` | `EXPLORATION` | `UNKNOWN`. Inferred as `IMPLEMENTATION` when files are modified but the field is UNKNOWN.

## Worktree

Git worktree, used to isolate ralph-tester runs from the main checkout so parallel test execution doesn't clobber files. Created fresh per invocation, cleaned up on completion.
