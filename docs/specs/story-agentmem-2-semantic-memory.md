# Story AGENTMEM-2: Codebase Pattern Memory (Semantic)

**Epic:** [Cross-Session Agent Memory](epic-agent-memory.md)
**Priority:** Medium
**Status:** Open
**Effort:** Medium
**Component:** new `.ralph/memory/project_index.json`, `.claude/hooks/on-session-start.sh`

---

## Problem

Every new Ralph session re-discovers project basics: framework, test runner, directory structure, coding conventions. This costs 1-3 explorer calls per session (~2000-6000 tokens) for information that changes infrequently.

## Solution

Generate a lightweight project index on first run (or when stale) that captures key project facts. Inject this index into the agent context to skip re-exploration.

## Implementation

### Step 1: Generate project index

```bash
ralph_generate_project_index() {
    local index_file="${RALPH_MEMORY_DIR}/project_index.json"
    mkdir -p "$RALPH_MEMORY_DIR"

    # Detect project characteristics (no LLM needed)
    local lang framework test_runner file_count

    # Language detection
    lang="unknown"
    [[ -f "package.json" ]] && lang="javascript/typescript"
    [[ -f "pyproject.toml" || -f "setup.py" ]] && lang="python"
    [[ -f "go.mod" ]] && lang="go"
    [[ -f "Cargo.toml" ]] && lang="rust"
    [[ -f "pom.xml" || -f "build.gradle" ]] && lang="java"

    # Test runner detection
    test_runner="unknown"
    [[ -f "jest.config.js" || -f "jest.config.ts" ]] && test_runner="jest"
    [[ -f "pytest.ini" || -f "pyproject.toml" ]] && grep -q "pytest" pyproject.toml 2>/dev/null && test_runner="pytest"
    [[ -f "package.json" ]] && grep -q "bats" package.json 2>/dev/null && test_runner="bats"

    # File count by type
    file_count=$(find . -name '.git' -prune -o -name 'node_modules' -prune -o -type f -print 2>/dev/null | wc -l)

    # Top-level directory structure
    local dirs
    dirs=$(ls -d */ 2>/dev/null | head -20 | tr '\n' ', ' | sed 's/,$//')

    # Key config files present
    local configs=""
    for f in .eslintrc* tsconfig.json .prettierrc* Makefile Dockerfile docker-compose.yml .github/workflows; do
        [[ -e "$f" ]] && configs="${configs}${f}, "
    done

    jq -n \
        --arg lang "$lang" \
        --arg test "$test_runner" \
        --argjson files "$file_count" \
        --arg dirs "$dirs" \
        --arg configs "${configs%, }" \
        --arg generated "$(date -u +%Y-%m-%dT%H:%M:%SZ)" \
        '{
            language: $lang,
            test_runner: $test,
            file_count: $files,
            top_directories: $dirs,
            config_files: $configs,
            generated_at: $generated
        }' > "$index_file"
}
```

### Step 2: Staleness check

```bash
ralph_is_index_stale() {
    local index_file="${RALPH_MEMORY_DIR}/project_index.json"
    [[ ! -f "$index_file" ]] && return 0  # Missing = stale

    local generated_at max_age_hours
    generated_at=$(jq -r '.generated_at' "$index_file" 2>/dev/null)
    max_age_hours=${RALPH_INDEX_MAX_AGE_HOURS:-24}

    # Compare timestamps
    local gen_epoch now_epoch
    gen_epoch=$(date -d "$generated_at" +%s 2>/dev/null || echo "0")
    now_epoch=$(date +%s)

    [[ $((now_epoch - gen_epoch)) -gt $((max_age_hours * 3600)) ]]
}
```

### Step 3: Inject into context

```bash
# In on-session-start.sh:
if [[ -f "${RALPH_MEMORY_DIR}/project_index.json" ]]; then
    echo "## Project Quick Reference"
    jq -r '"Language: \(.language)\nTest Runner: \(.test_runner)\nFile Count: \(.file_count)\nDirectories: \(.top_directories)\nConfig: \(.config_files)"' \
        "${RALPH_MEMORY_DIR}/project_index.json"
fi
```

## Design Notes

- **No LLM call**: Index generated by file detection heuristics. Fast, free, deterministic.
- **24-hour staleness**: Re-index daily or on explicit `ralph --reindex`. Project structure rarely changes faster.
- **~200 tokens**: The injected context is tiny compared to what explorer calls would cost.
- **Extensible**: Additional detectors (CI system, deployment target, database) can be added incrementally.

## Acceptance Criteria

- [ ] Project index generated on first run or when stale
- [ ] Language, test runner, directory structure, and config files detected
- [ ] Index injected into session context (~200 tokens)
- [ ] Staleness check prevents unnecessary re-indexing
- [ ] `ralph --reindex` forces re-generation
